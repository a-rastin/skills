# Source forensics

Treat a LaTeX project as a source graph, not a single text file. The inspection phase must discover every file and construct that can affect visible output.

## Root and build discovery

Search in this order:

1. explicit entry file supplied by the user;
2. files containing both `\documentclass` and `\begin{document}`;
3. `latexmkrc`, Makefile, shell scripts, CI, editor recipes, and README build commands;
4. generated wrappers that select chapters or modes.

Record the build command, working directory, engine, shell-escape requirement, bibliography tool, index/glossary tools, and environment variables. Multiple valid roots are distinct documents; ask which one to convert.

## Include graph

Resolve recursively:

- `\input{...}` and `\include{...}`;
- `\subfile{...}`;
- `\includeonly{...}`;
- package-local data files used by custom commands;
- `\bibliography`, `\addbibresource`, `\graphicspath`, `\includegraphics`;
- source files referenced by `minted`, `listings`, or `\verbatiminput`.

For each edge record: parent, child, command, source line, resolved path, existence, SHA-256, and whether it was traversed. Detect cycles and case mismatches. Resolve extensionless TeX paths using `.tex`; resolve graphics against the current file directory and every `\graphicspath` entry.

## Encoding and text integrity

- Detect BOM and decode as UTF-8 by default; use an explicit declared encoding only when evidence requires it.
- Normalize ordinary prose to NFC after parsing; preserve verbatim, listings, URLs, identifiers, and file paths byte-for-byte.
- Count Arabic-script characters and separately count Persian-specific code points (`پ چ ژ گ ی ک`) and source variants (`ي ك`). Report variants; transform them only under a user-approved normalization policy.
- Preserve ZWNJ U+200C, ZWJ U+200D, directional isolates, LRM/RLM, Persian/Arabic digits, and punctuation.
- Record every file whose decoded text contains replacement characters.

## Preamble inventory

Record:

- document class and options;
- geometry, paper size, margins, columns, orientation, line spacing;
- font packages and commands (`fontspec`, `xepersian`, `polyglossia`, `babel`, `bidi`, `\settextfont`, `\setlatintextfont`);
- heading, numbering, caption, footnote, header/footer, TOC, index, glossary, theorem, and bibliography packages;
- custom `\newcommand`, `\renewcommand`, `\DeclareMathOperator`, `\newenvironment`, and package-defined semantic macros;
- conditionals (`\if...`), counters, lengths, catcode changes, and shell-escape tools.

A macro is not “supported” because its name is recognized. Expand or transform it only after its argument contract and visible semantics are known.

## Direction and language inventory

Search for:

- XePersian and bidi package activation;
- `\lr`, `\rl`, `\textLR`, `\textRL`, `\LTR`, `\RTL`, `latin`, `persian`, `english`, `otherlanguage`;
- mixed-script headings, captions, tables, footnotes, headers, URLs, code, and bibliography entries;
- explicit font changes and Persian numeral commands.

Record whether direction is document-wide, environment-scoped, paragraph-scoped, or inline.

## Math inventory

Count and sample:

- inline `$...$`, `\(...\)`;
- display `\[...\]`, `equation`, `equation*`;
- `align`, `aligned`, `gather`, `multline`, `split`, `cases`, matrices, arrays;
- equation tags, labels, references, and subequations;
- custom math macros and operators;
- packages such as `amsmath`, `mathtools`, `unicode-math`, `siunitx`, `mhchem`, `physics`, and diagram packages.

Each custom macro used in math must be tested through Pandoc. Unsupported math becomes an explicit ledger item; the preferred repair is macro expansion into standard TeX, then OMML conversion.

## Table inventory

Identify `tabular`, `tabularx`, `longtable`, `array`, `supertabular`, `threeparttable`, `sidewaystable`, nested tables, and tables generated by macros. Record:

- logical rows/columns;
- column specifications and widths;
- `\multicolumn`, `\multirow`, merged headers, repeated headers;
- `booktabs` rules, vertical rules, color/shading, rotated cells;
- cell direction/language and math content;
- notes/captions/labels;
- resize/scale/landscape wrappers;
- whether the table can fit portrait Word margins at readable size.

## Figure and graph inventory

For every figure record the source command, caption, label, path, type, dimensions, crop/trim, rotation, subfigure group, and alternative text if present. Classify assets:

- direct raster: PNG/JPEG/TIFF;
- direct vector: SVG/EMF/WMF;
- PDF/EPS/PS requiring conversion;
- TikZ/PGFPlots/PSTricks/Asymptote/Graphviz/Mermaid source;
- generated plots with available data/code;
- missing or dynamically generated assets.

## Citations and references

Record citation package and commands, bibliography resources, style, sorting, locale, label/ref commands, hyperlinks, URL handling, footnotes/endnotes, indexes, glossaries, nomenclature, and acronym lists. Distinguish semantic citations from literal bracketed text.

## Raw and high-risk constructs

Flag:

- unknown environments or commands with visible arguments;
- `\special`, raw PDF literals, text positioning, overlays, absolute coordinates;
- PSTricks and shell-escape graphics;
- custom output routines, boxes, leaders, and page decorations;
- forms, annotations, embedded files, multimedia, or accessibility tags;
- commands inside verbatim-like contexts that a regex scanner may misread.

## Fidelity ledger schema

Every nontrivial construct receives an entry:

```json
{
  "id": "table:chapter2.tex:143",
  "kind": "table",
  "source_file": "chapter2.tex",
  "line_start": 143,
  "line_end": 171,
  "source_hash": "sha256:...",
  "summary": "RTL longtable with two header rows and multirow cells",
  "representation": "ooxml-patched",
  "editability": "native-table",
  "transform": "pandoc AST + gridSpan/vMerge + bidiVisual",
  "validation": ["row_count=18", "col_count=6", "caption_present"],
  "status": "planned",
  "warnings": []
}
```

## Inspection completion test

Inspection is complete only when:

- all reachable files are hashed;
- all dependencies resolve or are listed missing;
- every custom command/environment with visible output is classified;
- counts exist for sections, paragraphs, math, tables, figures, citations, labels, notes, and raw TeX;
- every high-risk construct has a ledger entry.
